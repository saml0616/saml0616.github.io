ēnk | Competitive Analysis

The competitive analysis looks at the three most popular note taking methods. For each competitor, I came up with features to implement or concepts to watch out for. Not every idea will be included in the final product as it may become overwhelming for the user to implement it all. This process helped me develop some of the features of the application.

OneNote

  1. Each page is organized into notebooks and then into sections.
  2. Implement a method to organize transcriptions.
  3. The drawing feature allows users to interact with OneNote like they would interact with a notebook.
  4. Allow for free drawing mode.
  5. The recording feature allows users to record as they type. A play button appears next to lines user records while typing.
  6. Allow users to jump to specific time stamps by clicking on transcribed sentences.
  7. Limited formatting options. There are very few variations you can do for each typed line. Help make the document more consistent.
  8. Implement strict formatting options, there is limited space on the phone so having preset options can save space.
  9. “Tags”/icons are very good visual cues and can also act as reminders of specific actions to take.
  10. Allow the user of tags/icons to use as shorthand for reminders students want to set during lectures.
  11. Bad formatting when printing from OneNote. The printed document often contains many empty pages and the text for each page is really big even if the font is set to 11.
  12. Format the document in such that users can easily print and review.

Pen and Paper

  1. Using pen and pencil to take notes is beneficial as you can quickly change from writing words to drawing diagrams.
  2. The transition from “highlighting” to drawing should be smooth.
  3. Challenge to reformat the document. Can’t move sections of text unless user rewrites.
  4. Allow users to edit order of text.
  5. Easy to misplace and lose paper documents.
  6. Allow saving on the app and exporting. Connect to Google Drive/Dropbox and automatically save the notes to the cloud.

    Microphone

  1. Transcribing audio frees up user’s attention to attend to more important things.
  2. Additional features of the app must not require too much attention from the user. The goal is to free up the user's attention and place it on processing lectured material.
  3. Users must go through the entire recording in order identify the useful information.
  4. The “highlight” feature should be accurate so users don’t have to look at the entire transcription.
  5. The recorded audio may not always be coherent, due to static or distance from the speaker.
  6. Implement a feature to compare with others to get a more accurate transcription of a lecture.
